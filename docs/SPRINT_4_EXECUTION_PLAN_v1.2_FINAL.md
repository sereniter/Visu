## Locked Technical Decisions

### Encoding Profile (v1)

```json
{
  "encoding_profile_version": "v1",
  "video_codec": "libx264",
  "pix_fmt": "yuv420p",
  "profile": "high",
  "preset": "medium",
  "crf": 18,
  "audio_codec": "aac",
  "audio_sample_rate": 48000
}
```

Stored in `config/default.json`. No overrides permitted.

### Audio Sample Rate Alignment

- Sprint 3 Piper output: **48000 Hz**
- Sprint 4 encoding profile: **48000 Hz**
- No resampling step. No hidden transformation. Determinism preserved.

### WebM → MP4 Is a Full Transcode

`raw.webm` from Playwright contains VP8 or VP9 video in a WebM container.  
Sprint 4 output is libx264 in MP4.  
This is a **full video transcode**, not a remux.

- Video stream is decoded and re-encoded via libx264 (CRF 18)
- `-c:v copy` is **never permitted**
- Audio streams are mixed and encoded as AAC at 48000 Hz

### Volume Ducking

```
music_ducking_db = -14 dB
```

- When narration present: music reduced by 14 dB
- When narration absent: music at 0 dB
- Value is fixed. Logged per run.

### FFmpeg Minimum Version

```
Minimum: 6.0
```

Hard fail if not installed or below 6.0. Version logged in RunContext.

### Music LUFS Governance

Music files must pass LUFS validation before merge:

- Acceptable range: **-15 to -17 LUFS**
- Measured via: `ffmpeg -i music.wav -filter_complex ebur128 -f null -`
- Out of range → hard fail
- Result logged per run

Music files stored in `assets/music/`. WAV only, 48000 Hz stereo, normalised offline.

### Fixture Governance

Integration test fixtures are static committed binaries:

- `raw_fixture.webm` ≤ 10 seconds, ≤ 5 MB
- `narration_fixture.wav` ≤ 10 seconds, ≤ 5 MB  
- `music_fixture.wav` ≤ 10 seconds, ≤ 5 MB
- Total committed fixture size ≤ 15 MB
- Git LFS not required under these constraints
- Fixtures must **not** be generated by Playwright during test runs

## Authoritative Operation Sequence

**Step 0 — FFmpeg Presence & Version Check**
- Run `ffmpeg -version`
- Extract semantic version
- Fail hard if not installed or version < 6.0
- Log `ffmpegVersion` in RunContext
- Abort immediately if check fails
- **Nothing else runs until this passes**

**Step 1 — Input Existence Validation**
- `rawVideoPath` exists
- `narrationPath` exists
- If `musicPath` provided → exists

**Step 2 — Music LUFS Validation (Conditional)**
```
if (musicPath != null):
  Run ebur128 filter
  Assert -15 to -17 LUFS
  Fail if out of range
  Log musicLufs value
else:
  Skip validation
  Log musicPath: null
  Log musicLufs: null
  Ducking disabled
```

**Step 3 — Duration Extraction (Input Gate)**
- `videoDurationMs` via `ffprobe` on `raw.webm`
- `narrationDurationMs` via `wav_utils` (Sprint 3 export)

**Step 4 — Drift Validation (Pre-Encode Input Gate)**

Drift is validated **before** encoding. It compares raw inputs only. It does **not** inspect `final.mp4`.

Rules:
```
narrationDurationMs ≤ videoDurationMs       → else fail immediately
(videoDurationMs - narrationDurationMs) ≤ 200ms  → else fail
```

- No padding
- No silence injection  
- No time stretching
- Hard stop on violation
- Log `driftMs`

**Step 5 — FFmpeg Transcode Execution**

Conceptual argument structure (deterministic, no shell string concatenation):

```
ffmpeg
  -i raw.webm
  -i narration.wav
  [-i music.wav]
  -filter_complex "[audio graph with ducking]"
  -map 0:v:0
  -map "[mixed_audio]"
  -c:v libx264
  -preset medium
  -profile:v high
  -pix_fmt yuv420p
  -crf 18
  -c:a aac
  -ar 48000
  -movflags +faststart
  final.mp4
```

Notes:
- Spawned via `child_process.spawn` — no shell string concatenation
- No dynamic flags
- No user overrides
- `-movflags +faststart` performs a second write pass (moov atom rewrite)
- Process must exit with code 0 before proceeding

**Step 6 — Output Validation**
- `final.mp4` exists
- File size > 0
- Duration > 0 (sanity check only)

**Step 7 — SHA256 Computation**
- Computed **after** `final.mp4` is fully written and finalised by FFmpeg
- Computed **after** `+faststart` rewrite completes (process exit code 0)
- No intermediate hash from stream pipe
- Log hash

**Step 8 — Metadata Construction**

```json
{
  "runId": "string",
  "encodingProfileVersion": "string",
  "ffmpegVersion": "string",
  "rawVideoPath": "string",
  "narrationPath": "string",
  "musicPath": "string | null",
  "musicLufs": "number | null",
  "durationMs": 12345,
  "avDriftMs": 0,
  "crf": 18,
  "audioSampleRate": 48000,
  "duckingDb": -14,
  "outputPath": "string",
  "outputSha256": "string",
  "generatedAt": "ISO8601"
}
```

**Step 9 — Metadata Schema Validation**
- Validate constructed object against `media_metadata_schema_v1.json`
- `additionalProperties: false`
- Fail if invalid

**Step 10 — Write + Log**
- Write `media_metadata.json`
- Log metadata hash

## Deliverables

### 4.1 FFmpeg Adapter (`src/adapters/ffmpeg_adapter.ts`)
- Deterministic argument builder (array-based, no shell concatenation)
- Explicit transcode — no stream copy
- Version detection and logging
- Encoding profile sourced from config
- Snapshot test for argument array

### 4.2 AV Merge Engine (`src/engines/av_merge_engine.ts`)
- Orchestrates full operation sequence (Steps 0–10)
- Accepts `rawVideoPath`, `narrationPath`, optional `musicPath`
- Returns `RunContext` with `artifacts.finalVideoPath` and `artifacts.metadataPath`
- Depends only on `FFmpegAdapter` interface — no direct FFmpeg imports in engine

### 4.3 AV Drift Validator (`src/validators/av_drift_validator.ts`)
- Isolated pure function
- Enforces directionality rule (narration ≤ video)
- Enforces delta rule (≤ 200ms)
- Returns `{ valid: boolean, driftMs: number, error?: string }`
- No correction logic

### 4.4 Music LUFS Validator (`src/validators/music_lufs_validator.ts`)
- Runs `ebur128` via FFmpeg
- Returns integrated LUFS value
- Asserts -15 to -17 range
- Accepts null input (skip branch)

### 4.5 Metadata Writer (`src/engines/metadata_writer.ts`)
- Computes SHA256 of `final.mp4`
- Constructs metadata object
- Validates against schema before writing
- Writes `media_metadata.json`
- Logs metadata hash

### 4.6 Schema (`schemas/media_metadata_schema_v1.json`)
- `additionalProperties: false`
- All fields typed and required (except nullable fields)

## Determinism Guarantees

Output is bit-identical across runs on the same machine given:

- Same `raw.webm`
- Same `narration.wav`
- Same `music.wav` (or both null)
- Same FFmpeg version (≥ 6.0)
- Same encoding profile (v1)
- Same CRF (18), preset (medium), sample rate (48000), ducking (-14 dB)

Cross-machine bit-identical output is **not guaranteed** — different FFmpeg builds may differ at the binary level. This is acceptable under current governance.
